What is EDIFACT?

A worldwide language for business documents

UN/EDIFACT — Electronic Data Interchange For Administration, Commerce and Transport — is the international standard behind orders, invoices, despatch advices and transport instructions. A file is plain text: lines called segments, each a tag plus +-separated elements. Hover a piece below to decode it.

NAD+CN+0410106314:160:Z12++Person Name+Street Nr 2+City2++12345+DE'
NADSegment tag — Name & Address. Maps to NADNameAddress.
CNParty qualifier — CN = Consignee. $nad->partyQualifier()
Person NameName — $nad->name()
Street Nr 2Street — $nad->street()
City2City — $nad->city()
DECountry — ISO 3166-1. $nad->countryCode()

The library turns that raw line into a typed object: no more counting + signs to find field 9. Empty elements (++) are preserved so positions never drift.

The envelope

Every interchange is a nested envelope

Optional UNG…UNE functional groups sit between the interchange and its messages — read them via $result->functionalGroups(). Trailer counts (UNT, UNZ) are computed for you when writing.

Why this library

One toolkit, the whole round-trip

Most parsers only read. This one models the full lifecycle of an interchange.

📥

Parse anything

Unknown tags degrade gracefully to raw values — no message type is unsupported. The parser never throws on unknown segments.

📤

Write it back

Serialize segments to valid .edi, or assemble a full UNB…UNZ interchange with auto-computed control counts.

✅

Validate

Pluggable rule sets for required segments, cardinality and relative order. Ready-made sets for ORDERS, INVOIC, DESADV, IFTMIN.

🌊

Stream

Parse multi-gigabyte files in bounded memory — one message at a time, honouring a leading UNA service advice.

🧱

Full envelope model

Interchange → groups → messages, with duplicate-preserving access and typed metadata on every envelope segment.

🏷️

32 typed segments

Domain accessors and qualifier constants out of the box — and trivially extensible with your own segment classes.

🔎

Fluent query API

Chain filters, transforms and pagination over every segment; a statistics analyzer aggregates counts and totals.

🌍

Charset-aware

UNOA…UNOY decoding to UTF-8, strict types, PSR-4, fully covered by PHPUnit, PHPStan, Psalm & Rector.

How it works

From raw .edi to typed objects

Pick a task. Every example is real, runnable code.

Parse a file or string into a typed ParserResult.
use EdifactParser\EdifactParser;

$result = EdifactParser::createWithDefaultSegments()
    ->parseFile('/path/to/order.edi'); // or ->parse($ediString)

foreach ($result->transactionMessages() as $message) {
    echo $message->messageType();   // 'IFTMIN', 'ORDERS', 'INVOIC', …
}

// Unknown segments never throw — read them via ->rawValues()
$result->functionalGroups();  // UNG…UNE groups, if any
$result->globalSegments();     // file-level UNA/UNB/UNZ
// Typed accessors — no magic array indices
$nad = $message->segmentByTagAndSubId('NAD', 'CN');
$nad?->name();          // 'Person Name'
$nad?->city();          // 'City2'
$nad?->countryCode();   // 'DE'  (ISO 3166-1)

// Numeric & date conversion built in
$qty?->quantityAsFloat();   // float
$pri?->priceAsFloat();      // float
$dtm?->asDateTime();        // DateTimeImmutable|null

// Line items group each LIN with its QTY/PRI/PIA details
foreach ($message->lineItems() as $item) {
    $item->segmentByTagAndSubId('QTY', '21')?->quantityAsFloat();
}
use EdifactParser\Segments\Qualifier\NADQualifier;

// Chain filters, transforms and pagination over every segment
$message->query()
    ->withTag('NAD')
    ->where(fn($s) => $s->partyQualifier() === NADQualifier::CONSIGNEE)
    ->limit(10)
    ->get();

$message->query()->withTag('NAD')->map(fn($s) => $s->name());
$message->query()->withTag('UNS')->exists();   // bool
$message->query()->ofType(NADNameAddress::class)->count();
use EdifactParser\Writer\InterchangeBuilder;
use EdifactParser\Writer\MessageBuilder;
use EdifactParser\Segments\NADNameAddress;
use EdifactParser\Segments\Qualifier\NADQualifier;

$nad = NADNameAddress::builder()
    ->withQualifier(NADQualifier::BUYER)
    ->withName('ACME Corporation')
    ->withCountryCode('US')
    ->build();

// UNT segment counts and UNZ control count filled in automatically
$edi = InterchangeBuilder::create('SENDER', 'RECIPIENT', 'REF1')
    ->addMessage(MessageBuilder::create('1', 'ORDERS')->addSegment($nad))
    ->toString();     // ready-to-send EDIFACT string
use EdifactParser\Validation\MessageRuleSet;
use EdifactParser\Validation\MessageValidator;

$rules = MessageRuleSet::forType('ORDERS')
    ->require('UNH', 'BGM', 'UNT')   // mandatory segments
    ->occurs('NAD', 1, 5)             // between 1 and 5
    ->inSequence('UNH', 'BGM', 'UNT');  // relative order

$validator = new MessageValidator();

foreach ($validator->validate($message, $rules) as $violation) {
    echo "{$violation->segmentTag()}: {$violation->message()}\n";
}
$validator->isValid($message, $rules);   // bool — never throws
use EdifactParser\Segments\AbstractSegment;

/** @psalm-immutable */
final class EQDEquipmentDetails extends AbstractSegment
{
    public function tag(): string { return 'EQD'; }

    // EQD+CN+ABCU1234567+22G1
    public function equipmentId(): string
    {
        return $this->firstComponent(2); // 'ABCU1234567'
    }
}

// Register alongside the 32 built-ins
$factory = SegmentFactory::withAdditionalSegments([
    'EQD' => EQDEquipmentDetails::class,
]);
